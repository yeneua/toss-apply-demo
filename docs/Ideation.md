고도화된 Headless UI 기반의 "디자인 시스템 라이브러리" 구축
토스는 자체 디자인 시스템인 **TDS(Toss Design System)**에 대한 자부심이 매우 높습니다. 컴포넌트의 재사용성뿐만 아니라, **Accessibility(접근성)**와 **DX(개발자 경험)**를 고려한 라이브러리를 직접 만들어보는 프로젝트입니다.

핵심 포인트
주제: Radix UI나 Headless UI처럼 로직은 살아있고 스타일은 자유로운 'Headless' 컴포넌트 5~7종 구축.

기술 스택: React, TypeScript, Tailwind CSS 또는 Vanilla Extract (토스가 최근 선호), 스토리북(Storybook).

UX Engineer 역량 증명:

복잡한 상태 관리: Compound Component Pattern을 사용하여 사용자가 유연하게 컴포넌트를 조립할 수 있게 설계.

접근성(WAI-ARIA): 스크린 리더 지원, 키보드 네비게이션(Focus Trapping 등) 완벽 구현.

애니메이션: Framer Motion을 활용해 토스 특유의 쫀득한 인터랙션(Micro-interactions) 추가.

토스가 매력을 느낄 포인트
"단순히 버튼을 만든 것이 아니라, 라이브러리를 사용하는 동료 개발자가 더 편하게 UI를 짤 수 있도록 추상화 수준을 고민했습니다."라는 점을 강조하세요.
